How We Found 11 Vulnerabilities in 30 Days Using AI-Augmented Analysis

The Christmas 2025 Research Sprint

Over the 2025 Christmas weekend, I discovered 11 critical vulnerabilities across two legacy healthcare and financial systems. Three received federal CVE assignments. Eight are pending MITRE review. This research was published by Packet Storm Security with staff endorsement.

This wasn't luck. It was systematic architectural analysis using AI augmentation. Here's the complete methodology and findings.

The AI-Augmented Methodology

Step 1: Target Selection (The "Abandonware Advantage")

Most security researchers target actively-maintained software with bug bounty programs. I target legacy systems—infrastructure that's too critical to replace but too complex to secure properly.

Why this works:

  • Legacy PHP applications built before modern security frameworks existed
  • Healthcare/financial systems where "it works" means "don't touch it"
  • No active security teams reviewing code
  • Downloaded thousands of times, embedded in real infrastructure

Target Profile: PHPGurukul Hospital Management System v4.0

Step 2: AI-Powered Code Analysis

I feed the entire codebase to Gemini and Claude with this prompt:

"Analyze this PHP application for architectural security failures. Focus on:
1. Authentication/authorization bypass opportunities
2. Business logic flaws (race conditions, state manipulation)
3. IDOR vulnerabilities in database queries
4. CSRF potential in state-changing operations
5. SQL injection in dynamic queries

Provide specific file paths and line numbers for each finding."

Why AI is critical:

  • Reviews 10,000+ lines of code in minutes
  • Identifies patterns humans miss (nested conditionals, logic branches)
  • Suggests exploit chains by connecting vulnerabilities

Step 3: Manual Verification (The 95% Effort)

AI finds potential vulnerabilities. I verify every single one manually:

  • Set up local testing environment (XAMPP, MySQL)
  • Reproduce the vulnerability with proof-of-concept code
  • Test exploit chains (can IDOR lead to privilege escalation?)
  • Document with screenshots, HTTP requests, database queries
  • Measure real-world impact (PHI exposure, financial manipulation)

Critical principle: 5% of time is hacking. 95% is documentation, verification, and bureaucracy.

Step 4: Responsible Disclosure

The Process:

  1. Attempt vendor contact (email, GitHub issues, LinkedIn)
  2. Wait 90 days for response (industry standard)
  3. If no response (common with abandonware), proceed to public disclosure
  4. Submit to MITRE for CVE assignment
  5. Publish to PacketStorm, GitHub with full technical documentation

Published CVEs (Federal Validation)

CVE-2025-70062: Cross-Site Request Forgery (CSRF)

CVSS 7.1 - Critical

Vulnerability Description

The PHPGurukul Hospital Management System v4.0 allows unauthenticated attackers to perform privileged actions on behalf of authenticated users through CSRF attacks.

Technical Details

Affected Endpoint: /hms/doctor/add-doctor.php

Vulnerability Type: Missing CSRF tokens on state-changing operations

Attack Vector:

  • Attacker crafts malicious HTML page with hidden form
  • Victim visits attacker's page while authenticated to HMS
  • Form auto-submits, creating new administrative account
  • Attacker gains persistent admin access

Impact

  • Unauthorized account creation with administrative privileges
  • Complete compromise of hospital management system
  • Access to patient medical records (PHI breach under HIPAA)

References

PacketStorm Advisory #213711 | GitHub PoC

CVE-2025-70063: Insecure Direct Object Reference (IDOR)

CVSS 7.5 - High

Vulnerability Description

Predictable patient ID parameters allow unauthorized access to medical records without authentication or authorization checks.

Technical Details

Affected Endpoint: /hms/view-patient.php?id=[PATIENT_ID]

Vulnerability Type: Missing access control validation

Attack Vector:

  • Patient IDs are sequential integers (1, 2, 3...)
  • No session validation on medical record viewing
  • Attacker enumerates IDs to access all patient records

Impact

  • Unauthorized access to Protected Health Information (PHI)
  • HIPAA violation - patient privacy breach
  • Data exfiltration of entire patient database possible

References

PacketStorm Advisory #213711 | GitHub PoC

CVE-2025-70064: Privilege Escalation (RBAC Bypass)

CVSS 9.1 - Critical

Vulnerability Description

Patient-level users can access full administrative dashboard without authentication through role-based access control failure.

Technical Details

Affected Component: Session management and role validation

Vulnerability Type: Improper access control

Attack Vector:

  • Patient authenticates with legitimate credentials
  • Session contains role information but lacks validation
  • Direct navigation to /admin/dashboard.php succeeds
  • Full administrative functions accessible

Impact

  • Complete system compromise from patient-level account
  • Ability to modify medical records, appointments, billing
  • Data deletion, user account manipulation
  • Horizontal privilege escalation to other patient records

Root Cause

Session validation checks user authentication but not role authorization. The application assumes URL-based access control is sufficient.

Key Takeaways

1. Legacy Systems Are Gold Mines

Abandonware deployed in real infrastructure offers more learning value than actively-defended bug bounty targets.

2. AI Accelerates Discovery, Not Replacement

AI finds patterns. Humans verify impact. The combination is exponentially more effective than either alone.

3. Documentation > Exploitation

Finding a vulnerability takes hours. Documenting it properly for CVE assignment takes days. Master the bureaucracy.

4. Zero Cost, Maximum Impact

This research cost ₹0. No expensive tools, no bug bounty fees, no certifications required. Just methodology and discipline.
