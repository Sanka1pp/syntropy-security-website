How I Found 14 Zero-Days Over Christmas

Technical analysis of 10 federal CVE assignments

Executive Summary

Over Christmas weekend 2025, I discovered 14 critical vulnerabilities across two legacy systems using AI-augmented analysis. Ten received federal CVE assignments from MITRE (71% success rate). This report details the methodology and findings.

The Results

CVE Assignments Received:

• CVE-2025-70062 (Cross-Site Request Forgery)
• CVE-2025-70063 (Insecure Direct Object Reference)
• CVE-2025-70064 (Privilege Escalation)
• CVE-2026-26495 (SQL Injection - Auth Bypass)
• CVE-2026-26497 (Stored Cross-Site Scripting)
• CVE-2026-26499 (IDOR - Admin Dashboard)
• CVE-2026-26500 (Authenticated SQL Injection → RCE)
• CVE-2026-26501 (Integer Overflow)
• CVE-2026-26502 (Race Condition - Double Spending)
• CVE-2026-26503 (Stored XSS - Admin Takeover)

The Methodology

Traditional pentesting is 5% exploitation and 95% reconnaissance. I inverted this using AI-augmented architectural analysis.

Phase 1: Target Selection
I targeted legacy PHP applications in healthcare and finance—systems abandoned by vendors but still deployed in critical infrastructure.

Phase 2: AI-Augmented Analysis
I uploaded entire codebases to Claude and Gemini with a single prompt: "Analyze for architectural security failures." Within minutes, the AI identified 40+ potential vulnerabilities.

Phase 3: Manual Verification
I manually verified each finding, developed proof-of-concept exploits, and documented the attack chains.

Phase 4: Responsible Disclosure
I coordinated with MITRE for CVE assignments and published advisories through Packet Storm Security.

Key Findings

The most critical vulnerabilities weren't in code—they were in unvalidated assumptions:

• Applications assumed users would only request their own data (IDOR)
• Applications assumed session variables came from legitimate sources (Privilege Escalation)
• Applications assumed requests were intentional (CSRF)
• Applications assumed negative values were impossible (Integer Overflow)
